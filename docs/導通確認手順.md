# RTG Coding Agent — Cloud Run 導通確認手順

`rtg-coding-agent` を Cloud Run（東京リージョン・IAM 認証あり）にデプロイした後、
手動で疎通を確認するための手順。

## デプロイ済みサービス情報

| 項目 | 値 |
|---|---|
| プロジェクト | `reverse-turing-demo-suzuki` |
| リージョン | `asia-northeast1`（東京） |
| サービス名 | `rtg-coding-agent` |
| Service URL | `https://rtg-coding-agent-947565277742.asia-northeast1.run.app` |
| 認証 | IAM 認証必須（`--no-allow-unauthenticated`）。`roles/run.invoker` を持つ ID のみアクセス可 |
| ランタイム SA | `rtg-coding-agent-sa@reverse-turing-demo-suzuki.iam.gserviceaccount.com` |

> Service URL は再デプロイしても変わらない。忘れた場合は以下で再取得できる。
> ```bash
> gcloud run services describe rtg-coding-agent \
>   --region=asia-northeast1 \
>   --format='value(status.url)' \
>   --project=reverse-turing-demo-suzuki
> ```

## 前提

- `gcloud auth login` 済みで、かつ自アカウントに当該サービスの `roles/run.invoker` が付与されていること（未付与の場合は下記コマンドで付与）。

```bash
gcloud run services add-iam-policy-binding rtg-coding-agent \
  --region=asia-northeast1 \
  --member="user:<自分のGoogleアカウント>" \
  --role="roles/run.invoker" \
  --project=reverse-turing-demo-suzuki
```

## 手順

### 1. 認証なしアクセスが拒否されることを確認（403 が正常）

IAM 認証が有効になっていることの確認。

```bash
SERVICE_URL="https://rtg-coding-agent-947565277742.asia-northeast1.run.app"
curl -s -o /dev/null -w "HTTP %{http_code}\n" "${SERVICE_URL}/"
# → HTTP 403 が返れば正常（認証必須の設定が効いている）
```

### 2. ID トークンを付与した軽量疎通確認（200 が正常）

FastAPI が正常に起動し、アプリが応答できる状態かを確認する。

```bash
TOKEN=$(gcloud auth print-identity-token)
curl -s -o /dev/null -w "HTTP %{http_code}\n" \
  -H "Authorization: Bearer ${TOKEN}" \
  "${SERVICE_URL}/openapi.json"
# → HTTP 200 が返れば正常
```

### 3. A2A 本体エンドポイントの疎通確認

実際に A2A JSON-RPC 2.0 リクエストを送り、Vertex AI 経由で spec が生成されることを確認する。

**注意（Windows/Git Bash 環境）**: 日本語を含む JSON を `curl -d '...'` で直接渡すと
シェルの改行・エンコーディング処理により `There was an error parsing the body`（HTTP 400）が
発生することがある。その場合は JSON を UTF-8 ファイルに保存し `--data-binary @<file>` で送ること。

```bash
cat > /tmp/a2a_request.json << 'EOF'
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "message/send",
  "params": {
    "message": {
      "role": "user",
      "parts": [{"text": "以下の要望を実現する spec ファイル（Markdown・実現方針/変更範囲/手順）を作成してください:\nカート画面にクーポン機能を追加したい"}]
    }
  }
}
EOF

TOKEN=$(gcloud auth print-identity-token)
curl -s -X POST "${SERVICE_URL}/" \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json; charset=utf-8" \
  --data-binary @/tmp/a2a_request.json \
  -w "\n\nHTTP %{http_code}\n"
```

**期待結果**: HTTP 200 で、以下の形式のレスポンスが返る。

```json
{
  "result": {
    "artifacts": [
      { "parts": [{ "text": "## 実現方針\n...\n## 変更範囲\n...\n## 手順\n..." }] }
    ]
  }
}
```

### 4. ログでエラーが出ていないか確認

```bash
gcloud run services logs read rtg-coding-agent \
  --region=asia-northeast1 \
  --project=reverse-turing-demo-suzuki \
  --limit=50
```

`INFO:     Application startup complete.` の後、リクエストごとに
`GET`/`POST` のステータスコードが記録される。500番台エラーや
`エージェント実行エラー` のログが出ていないことを確認する。

## トラブルシューティング

| 症状 | 原因の可能性 | 対処 |
|---|---|---|
| 手順1で 403 でなく 200 | 未認証公開（`--allow-unauthenticated`）になっている | `gcloud run services update rtg-coding-agent --region=asia-northeast1 --no-allow-unauthenticated` |
| 手順2/3で 403 | 自アカウントに `run.invoker` が無い、または ID トークンが期限切れ | 前提セクションのコマンドで権限付与、`TOKEN` を再取得 |
| 手順3で 400 (`There was an error parsing the body`) | シェル経由の JSON エンコーディング崩れ | ファイル化して `--data-binary @file` で送信 |
| 手順3で 502 | Vertex AI 呼び出し失敗（権限不足・タイムアウト等） | ログでスタックトレースを確認。ランタイム SA に `roles/aiplatform.user` があるか確認 |
| 手順3で 500 | エージェントがタイムアウト（120秒）または応答が空 | ログで `run_query` 側の例外内容を確認 |
