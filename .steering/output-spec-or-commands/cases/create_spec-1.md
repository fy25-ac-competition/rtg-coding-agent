カート画面にクーポン機能を追加したい

## 出力全文

## 実現方針
カート画面にクーポン機能を追加するため、フロントエンド、バックエンド、データベースにわたる包括的な改修を行います。ユーザーはカート画面でクーポンコードを入力し、適用ボタンを押すことで、注文合計金額に割引が適用されるようにします。バックエンドではクーポンの有効性チェックと割引計算を行い、データベースにはクーポン情報と利用履歴を保存します。

## 変更範囲
1.  **データベース**:
    *   `coupons` テーブルの新規作成（例: `id`, `code`, `type` (定額/定率), `value`, `min_purchase_amount`, `expiry_date`, `usage_limit`, `used_count`, `created_at`, `updated_at`）。
    *   `orders` テーブルに `coupon_id` (外部キー) および `discount_amount` カラムを追加。
2.  **バックエンド (API)**:
    *   クーポンコードを検証し、割引を適用するための新しいAPIエンドポイント（例: `POST /api/cart/apply-coupon`）を追加。
    *   カート情報取得API（例: `GET /api/cart`）を修正し、適用済みのクーポン情報と割引額を返すようにする。
    *   注文作成API（例: `POST /api/order`）を修正し、カートに適用されているクーポンと割引額を注文に紐付けるロジックを追加。
3.  **バックエンド (ビジネスロジック)**:
    *   クーポンコードの構文チェック、有効期限チェック、利用回数上限チェック、最低購入金額条件チェックなどの検証ロジックを実装。
    *   クーポンのタイプ（定額、定率）に応じた割引額計算ロジックを実装。
    *   クーポン利用後の `used_count` 更新ロジックを実装。
4.  **フロントエンド (カート画面)**:
    *   カート画面にクーポンコード入力フォームと「適用」ボタンを追加。
    *   「適用」ボタン押下時に、バックエンドのクーポン適用APIを呼び出すJavaScriptロジックを実装。
    *   APIからのレスポンスに基づき、カート合計金額の内訳（小計、割引額、合計）を動的に更新して表示。
    *   クーポン適用成功/失敗のメッセージ表示機能。

## 手順
1.  **データベーススキーマの変更**:
    *   `coupons` テーブルの作成SQLを実行。
    *   `orders` テーブルへの `coupon_id` および `discount_amount` カラム追加のマイグレーションを実行。
2.  **バックエンド実装**:
    *   クーポンを管理するモデル（ORM）を定義。
    *   クーポン適用ロジックをカプセル化したサービス層またはユースケース層のクラスを作成。
    *   クーポン適用APIエンドポイントの実装。リクエストからクーポンコードを受け取り、クーポン適用ロジックを呼び出して結果を返す。
    *   カート情報取得APIの修正。適用中のクーポンがあればその情報をレスポンスに含める。
    *   注文作成APIの修正。注文永続化時にクーポン情報を紐付け、割引額を保存する。
3.  **フロントエンド実装**:
    *   カート画面のHTMLテンプレートを修正し、クーポン入力フォームと適用ボタンを追加。
    *   JavaScriptコードを記述し、以下の処理を実装:
        *   入力フォームの値を取得。
        *   「適用」ボタンクリックイベントで、上記で実装したバックエンドのクーポン適用APIをFetch APIなどで呼び出す。
        *   APIの成功レスポンスに基づいて、画面上の合計金額表示を更新し、割引額を明示する。
        *   APIの失敗レスポンスに応じて、エラーメッセージを表示する。
    *   クーポン適用後の画面遷移やユーザー体験を考慮したUI/UX調整。
4.  **テスト**:
    *   単体テスト: クーポン検証ロジック、割引計算ロジック。
    *   結合テスト: フロントエンドからバックエンドAPIを介してクーポンが正しく適用され、合計金額が更新されるか。
    *   シナリオテスト: 有効なクーポン、無効なクーポン、期限切れクーポン、利用回数超過クーポンなど、様々なパターンでの動作確認。## 実現方針
